## 2. Authentication

Every request needs **two** things:

1. A **bearer API key** in the `Authorization` header: `Authorization: Bearer sk_car_wjKCTeSjr1p8JcsDKYtXMp`
2. The **API version** header: `Cartesia-Version: 2026-03-01` (current enum; the SDK defaults to `2025-11-04` but the docs now show `2026-03-01`).

```bash
curl https://api.cartesia.ai/voices?limit=3 \
  -H "Authorization: Bearer sk_car_wjKCTeSjr1p8JcsDKYtXMp" \
  -H "Cartesia-Version: 2026-03-01"
```

**Never expose the `sk_car_…` key in the browser.** For client-side calls (e.g. a
direct browser→Cartesia WebSocket), mint a short-lived access token server-side via
`POST /access-token` (max 1 hour, `expires_in` in seconds) and use that as the bearer
token instead. See §6.

## 4. The three TTS endpoints

All three use the **same** `model_id`, `voice`, and `output_format` parameters. They differ
in wire shape, whether you can stream input, and whether you get timestamps.

| | Multiple generations / connection | Timestamps | Continuations (stream input) |
|---|---|---|---|
| **WebSocket** | Yes | Yes | Yes |
| **Bytes** | No (one POST per generation) | No | No |
| **SSE** | No (one POST per generation) | Yes | No |

**Decision tree:**
- Full transcript known up front, just want audio → **Bytes**
- Full transcript, need word/phoneme timestamps over HTTP → **SSE**
- Text arrives incrementally (e.g. from an LLM), lowest latency across turns → **WebSocket**

### 4.1 `POST /tts/bytes` — streaming HTTP body (simplest, recommended for Next.js)

**Request body (JSON):**
```jsonc
{
  "model_id": "sonic-3.5",
  "transcript": "Hello, world! I'm generating audio on Cartesia.",
  "voice": { "mode": "id", "id": "db6b0ed5-d5d3-463d-ae85-518a07d3c2b4" },
  "output_format": { "container": "wav", "encoding": "pcm_s16le", "sample_rate": 24000 },
  "language": "en",                       // optional, defaults per voice
  "generation_config": {                  // optional, sonic-3 / sonic-3.5 only
    "volume": 1,        // 0.5 – 2.0
    "speed": 1,         // 0.6 – 1.5
    "emotion": "neutral" // e.g. calm, angry, content, sad, scared, happy, excited…
  }
}
```

**Response:** the body is the audio stream (`audio/wav`, `audio/mpeg`, or raw PCM). No JSON
wrapper. `Content-Type` reflects the container. This is the cleanest format to proxy from a
Next.js route — you can stream the body straight through.

> ✅ **Verified live:** `POST /tts/bytes` with the key above returned `200`,
> `Content-Type: audio/wav`, a 49,714-byte body beginning with `RIFF…WAVEfmt `
> (PCM 16-bit, mono, 24000 Hz).

### 4.2 `POST /tts/sse` — Server-Sent Events with timestamps

Same request body as bytes, **plus** `add_timestamps: true` and `output_format` restricted
to `raw` (PCM only). The response is a `text/event-stream`. Each frame is
`data: <json>\n\n`. Event `type`s:

- `chunk` — `{ type, done:false, status_code:206, step_time, context_id, data:"<base64 audio>" }`
- `timestamps` — `{ word_timestamps:{ words:[], start:[], end:[] } }`
- `phoneme_timestamps` — phoneme-level timing
- `done` — `{ type:"done", done:true, status_code:206 }`
- `error` — `{ type:"error", title, message, error_code, request_id, doc_url }`

Audio arrives base64-encoded inside `data`. SSE is best when you specifically need
word/phoneme timing over plain HTTP.

### 4.3 `wss://api.cartesia.ai/tts/websocket` — realtime, streaming input

Connect with the API key as a query param or header, then send JSON messages. Each message
belongs to a **context** identified by `context_id`. You can run many contexts on one socket.

**Send — generation request:**
```jsonc
{
  "model_id": "sonic-latest",
  "transcript": "Hello, world! I'm generating audio on Cartesia!",
  "voice": { "mode": "id", "id": "a0e99841-438c-4a64-b679-ae501e7d6091" },
  "language": "en",
  "context_id": "ab977222-f9e0-4563-a1c0-5a934ae8fdd6",
  "output_format": { "container": "raw", "encoding": "pcm_s16le", "sample_rate": 8000 },
  "add_timestamps": true,
  "continue": false          // false = this is the final chunk for this utterance
}
```

**Continuations:** for streaming LLM output, send multiple messages sharing one `context_id`,
with `continue: true` on partials and `continue: false` on the last part. The server
concatenates them and preserves prosody across chunks.

**Cancel a context:** `{ "context_id": "…", "cancel": true }`

**Receive** messages with the same `context_id`, of type `chunk`, `timestamps`,
`phoneme_timestamps`, `error`, or `done`. Audio in `chunk.data` is base64-encoded PCM.

**Timeouts:** idle WebSocket connections are closed after **5 minutes**. Reconnect when idle.
**Connection limit:** 10× your concurrency limit (e.g. Scale plan = 15 concurrency → 150 sockets).

#### WebSocket via the SDK (Node)

```ts
import Cartesia from "@cartesia/cartesia-js";
const client = new Cartesia({ apiKey: process.env.CARTESIA_API_KEY! });

const ws = await client.tts.websocket();
const ctx = ws.context({
  model_id: "sonic-3.5",
  voice: { mode: "id", id: "f786b574-daa5-4673-aa0c-cbe3e8534c02" }, // Katie
  output_format: { container: "raw", encoding: "pcm_f32le", sample_rate: 44100 },
});

for (const part of ["Hi there! ", "Welcome to ", "Cartesia Sonic."]) {
  await ctx.push({ transcript: part }); // continue: true by default
}
await ctx.no_more_inputs();              // sends continue: false

for await (const event of ctx.receive()) {
  if (event.type === "chunk" && event.audio) {
    // event.audio is a Uint8Array of PCM bytes
  } else if (event.type === "done") break;
}
ws.close();
```

> In the browser, the native `WebSocket` API is enough — don't install `ws`.

## 5. Next.js API route integration (drop-in for THIS project)

This repo already has `src/app/api/tts/route.ts` (currently ElevenLabs → z-ai fallback) and a
`useVoice.ts` hook that does `fetch("/api/tts", { body: { text } })` then plays the returned
audio blob via `new Audio()`. **Cartesia slots in perfectly** because the hook only needs an
audio blob — and Cartesia `/tts/bytes` returns exactly that.

### Option A — raw `fetch` (no extra dependency), streaming passthrough

```ts
// src/app/api/tts/route.ts  (Cartesia version)
import { NextRequest, NextResponse } from "next/server";

export const runtime = "nodejs";
export const maxDuration = 30;

const CARTESIA_API = "https://api.cartesia.ai";
const CARTESIA_VERSION = "2026-03-01";

// Default voice: "Skylar - Friendly Guide" (verified ID from GET /voices)
const DEFAULT_VOICE_ID = "db6b0ed5-d5d3-463d-ae85-518a07d3c2b4";

export async function POST(req: NextRequest) {
  let body: { text: string; voice?: string; emotion?: string };
  try {
    body = await req.json();
  } catch {
    return NextResponse.json({ error: "Invalid JSON" }, { status: 400 });
  }

  const text = (body.text ?? "").trim();
  if (!text) return NextResponse.json({ error: "Empty text" }, { status: 400 });

  const apiKey = process.env.CARTESIA_API_KEY;
  if (!apiKey) return NextResponse.json({ error: "CARTESIA_API_KEY not set" }, { status: 500 });

  const upstream = await fetch(`${CARTESIA_API}/tts/bytes`, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${apiKey}`,
      "Cartesia-Version": CARTESIA_VERSION,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model_id: "sonic-3.5",
      transcript: text,
      voice: { mode: "id", id: body.voice || DEFAULT_VOICE_ID },
      output_format: { container: "wav", encoding: "pcm_s16le", sample_rate: 24000 },
      ...(body.emotion ? { generation_config: { emotion: body.emotion } } : {}),
    }),
  });

  if (!upstream.ok || !upstream.body) {
    const detail = await upstream.text().catch(() => "");
    return NextResponse.json(
      { error: `Cartesia ${upstream.status}`, detail },
      { status: upstream.status }
    );
  }

  // Stream the audio body straight through to the browser.
  return new NextResponse(upstream.body, {
    headers: {
      "Content-Type": upstream.headers.get("content-type") || "audio/wav",
      "Cache-Control": "no-store",
    },
  });
}
```

Add to `.env`:
```
CARTESIA_API_KEY=sk_car_wjKCTeSjr1p8JcsDKYtXMp
```

The existing `useVoice.ts` `speakWithServerTTS()` already does `res.blob()` → `new Audio(url)`
→ `audio.play()`. A WAV response works with **zero** client changes. (If you prefer MP3 to
shrink payload size, change `container` to `"mp3"` and add `bit_rate: 128000`; the `Audio`
element plays MP3 natively too.)

### Option B — using the SDK

```ts
// src/lib/cartesia.ts
import Cartesia from "@cartesia/cartesia-js";

let _client: Cartesia | null = null;
export function getCartesia() {
  if (!_client) _client = new Cartesia({ apiKey: process.env.CARTESIA_API_KEY! });
  return _client;
}

// src/app/api/tts/route.ts
import { NextRequest, NextResponse } from "next/server";
import { getCartesia } from "@/lib/cartesia";

export const runtime = "nodejs";
export const maxDuration = 30;

export async function POST(req: NextRequest) {
  const { text, voice, emotion } = await req.json();
  if (!text?.trim()) return NextResponse.json({ error: "Empty text" }, { status: 400 });

  const client = getCartesia();
  const response = await client.tts.generate({
    model_id: "sonic-3.5",
    transcript: text,
    voice: { mode: "id", id: voice || "db6b0ed5-d5d3-463d-ae85-518a07d3c2b4" },
    output_format: { container: "wav", encoding: "pcm_s16le", sample_rate: 24000 },
    ...(emotion ? { generation_config: { emotion } } : {}),
  });

  const buf = Buffer.from(await response.arrayBuffer());
  return new NextResponse(buf, {
    headers: { "Content-Type": "audio/wav", "Content-Length": String(buf.length) },
  });
}
```

## 6. Calling Cartesia directly from the browser (low-latency WebSocket)

For true realtime (LLM token streaming → audio), connect the browser straight to Cartesia's
WebSocket. **Do not** ship the `sk_car_` key to the browser. Instead:

1. **Server route** mints a short-lived token:
   ```ts
   // src/app/api/cartesia-token/route.ts
   import { NextResponse } from "next/server";
   import { getCartesia } from "@/lib/cartesia";

   export const runtime = "nodejs";
   export async function GET() {
     const client = getCartesia();
     // SDK helper (or POST /access-token yourself)
     const { token } = await client.auth.accessToken({
       expires_in: 600,            // 10 minutes
       grants: { tts: true },      // only the capabilities you need
     });
     return NextResponse.json({ token });
   }
   ```
2. **Client** fetches the token, then opens the WebSocket:
   ```ts
   const { token } = await fetch("/api/cartesia-token").then(r => r.json());
   const ws = new WebSocket(
     `wss://api.cartesia.ai/tts/websocket?api_key=${encodeURIComponent(token)}&cartesia_version=2026-03-01`
   );
   ws.onopen = () => ws.send(JSON.stringify({
     model_id: "sonic-3.5",
     transcript: "Streaming straight from the browser!",
     voice: { mode: "id", id: "db6b0ed5-d5d3-463d-ae85-518a07d3c2b4" },
     context_id: crypto.randomUUID(),
     output_format: { container: "raw", encoding: "pcm_f32le", sample_rate: 44100 },
   ```

## 7. Voice cloning

Cartesia offers **two** cloning tiers. Instant cloning works on Pro+; Pro Voice Cloning needs
Startup+.

| Feature | Required audio | Cost to create | Cost to use for TTS | Min plan |
|---|---|---|---|---|
| **Instant Voice Clone** | ≤10 s clip | Free | 1 credit/char | Pro ($5/mo) |
| **Pro Voice Clone** | ≥30 min (2 h recommended) | 1M credits on success | 1.5 credits/char | Startup ($49/mo) |

### 7.1 Instant Voice Clone — `POST /voices/clone`

Multipart form upload. Supported clip formats: `flac, mp3, mpeg, mpga, oga, ogg, wav, webm`.
Best practices: clear single speaker, minimal background noise, no long pauses, trimmed,
spoken in the target language.

```ts
// Server-side (Node) — uses the SDK's toFile helper
import fs from "fs";
import Cartesia, { toFile } from "@cartesia/cartesia-js";

const client = new Cartesia({ apiKey: process.env.CARTESIA_API_KEY! });

const voice = await client.voices.clone({
  clip: await toFile(fs.createReadStream("sample.wav"), "sample.wav"),
  name: "My Custom Voice",
  description: "Cloned from a 9-second sample",
  language: "en",
  // access: { type: "private" },        // "private" (default) or "public"
  // base_voice_id: "...",                // optional base voice to derive from
});

console.log(voice.id); // use this id in voice.id for /tts/bytes etc.
```

Equivalent raw `curl`:
```bash
curl -X POST https://api.cartesia.ai/voices/clone \
  -H "Authorization: Bearer sk_car_..." \
  -H "Cartesia-Version: 2026-03-01" \
  -F "clip=@sample.wav" \
  -F "name=My Custom Voice" \
  -F "language=en"
```

**Response (`VoiceMetadata`):** `{ id, user_id, access:{type,visibility}, name, description,
created_at, language, is_public }`. Save `voice.id` and pass it as `voice.id` in TTS calls.

### 7.2 Pro Voice Clone — datasets + fine-tunes

Pro cloning fine-tunes a model on your data, then creates 4 voices from selected clips. Flow:

1. **`POST /datasets`** — `{ name, description }` → dataset id
2. **`POST /datasets/{id}/files`** — multipart upload of `.wav/.mp3/.flac/.ogg/...` (min 30 min
   total; 2 h recommended). Field `purpose: "fine_tune"`.
3. **`POST /fine-tunes`** — `{ name, description, language, model_id, dataset }` → fine-tune id
   (e.g. `model_id: "sonic-2"`).
4. **Poll `GET /fine-tunes/{id}`** until `status === "completed"` (~3 hours; only charged on
   success — 1M credits).
5. **`GET /fine-tunes/{id}/voices`** → the 4 generated voice ids. Use them in TTS calls; they
   auto-route to your fine-tuned model.

Full Python script is in the official docs; the same endpoints work from Node with `fetch` or
the SDK (`client.datasets.create()`, `client.datasets.files.upload()`,
`client.fineTunes.create()`, `client.fineTunes.listVoices()`).

## 9. Output format reference

`output_format` has three knobs: `container`, `encoding`, `sample_rate`.

| Container | Where supported | Notes |
|---|---|---|
| `wav` | **bytes only** | Self-describing; best for file saving / `<audio>` playback |
| `mp3` | **bytes only** | Smallest payload; needs `bit_rate` (32k–192k) + `sample_rate` |
| `raw` | bytes, SSE, WebSocket | Just PCM; match encoding/sample_rate to your player |

**Encodings:**
| Encoding | Bit depth | Typical use | Sample rate |
|---|---|---|---|
| `pcm_mulaw` | 8-bit µ-law | Twilio / NA-JP telephony | 8000 |
| `pcm_alaw` | 8-bit A-law | European telephony | 8000 |
| `pcm_s16le` | 16-bit int | LiveKit, Pipecat, voice agents | 16000 |
| `pcm_f32le` | 32-bit float | Web Audio API / browsers | 44100 or 48000 |

For this project's `/api/tts` → `<audio>` element, **WAV `pcm_s16le` 24 kHz** (verified working)
is the simplest. Switch to **MP3 128k** if you want smaller responses over the wire.

## 11. Models & latency

| Model | Status | Languages | Latency | Notes |
|---|---|---|---|---|
| `sonic-3.5` | **Stable / recommended** | 42 | **sub-90 ms TTFB** | #1 naturalness; alphanumerics + heteronyms correct w/o preprocessing |
| `sonic-3` | Stable | 42 | ~similar | Predecessor; pin `sonic-3-2026-01-12` for reproducibility |
| `sonic-latest` | Beta | 42 | latest | Testing only; not for production |
| `sonic-2` | Stable | 8 (en,fr,de,es,pt,zh,ja,ko) | — | Older; still served |
| `sonic-turbo` | Stable (legacy) | 8 | historically ~40 ms TTFA | **Not in the current `model_id` enum for bytes/SSE/WS** — superseded by sonic-3.5 |
| `sonic` | Sunsetted | 13 | — | Don't use |

**On "Sonic Turbo":** the original `sonic-turbo` model (snapshot `sonic-turbo-2025-06-04`) is an
older, lower-language-count model. The current API's `TTSModelID` enum only accepts
`sonic-3.5`, `sonic-3`, and `sonic-latest`. Independent benchmarks put Cartesia's realtime
TTFB at **~80 ms** (vs ElevenLabs Turbo ~250–400 ms) and as low as **40 ms TTFA** in some
setups. In practice sonic-3.5 gives you the best of both — fastest *and* most natural.

Pin a dated snapshot (`sonic-3.5-2026-05-04`) if you need reproducible evals; use the alias
`sonic-3.5` to auto-track the latest stable.

## 12. Concurrency & limits

| Plan | Concurrent TTS contexts | Max WebSocket connections (10×) |
|---|---|---|
| Free | 2 | 20 |
| Pro | 3 | 30 |
| Startup | 5 | 50 |
| Scale | 15 | 150 |
| Enterprise | custom | custom |

- Each HTTP request = 1 context. On WebSocket, a unique `context_id` = 1 context (reusing the
  same id across messages does **not** add to your count).
- Exceeding → `429 Too Many Requests`.
- Idle TTS WebSockets closed after **5 minutes**; idle STT sockets after 3 minutes.
- For conversational agents, ~4× the concurrency limit in parallel conversations is a rule of
  thumb (e.g. 15 concurrency ≈ 60 simultaneous conversations).

## 13. Error handling

| HTTP status | Error class | Meaning |
|---|---|---|
| 400 | BadRequestError | Malformed request / invalid model |
| 401 | AuthenticationError | Bad/missing API key |
| 403 | PermissionDeniedError | Key lacks scope (e.g. cloning on Free plan) |
| 404 | NotFoundError | Unknown voice/resource id |
| 422 | UnprocessableEntityError | Schema validation failure |
| 429 | RateLimitError | Concurrency/WS limit exceeded |
| ≥500 | InternalServerError | Cartesia-side fault (auto-retried) |

The SDK auto-retries connection errors, 408, 409, 429, and ≥500 (2× by default, exponential
backoff). Configure with `new Cartesia({ maxRetries: 0, timeout: 20000 })`.
